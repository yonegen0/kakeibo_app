# AI分析機能付き家計簿アプリ

## フロント＋バックエンド統合 詳細設計書（AWS版・完全版）

---

## 1. 設計方針（最重要）

## 1.1 アーキテクチャ思想

### Before（現行）

- mockStore中心
- Next.js API内で完結
- 同期処理

### After（AWS版）

- API Gateway + Lambda + 非同期処理
- psvId中心のデータ管理
- スケーラブルなイベント駆動設計

---

## 1.2 設計原則

- 主キー：`psvId` / `reportId`
- CSV処理：非同期
- AI分析：同期（初期）
- データ：S3 + DynamoDB分離
- API：責務ごとにLambda分割

---

## 2. 全体アーキテクチャ

```mermaid
graph TD
    U[User] --> CF[CloudFront]
    CF --> S3H[S3 Hosting]

    U --> APIGW[API Gateway]

    APIGW --> L1[Lambda: PSV]
    APIGW --> L2[Lambda: Summary]
    APIGW --> L3[Lambda: Analyze]
    APIGW --> L4[Lambda: Report]

    L1 --> S3D[S3 Raw Data]
    L1 --> SQS[SQS]

    SQS --> L_ETL[Lambda: ETL]
    L_ETL --> DDB[(DynamoDB)]

    L2 --> DDB
    L3 --> DDB
    L3 --> Bedrock
    L4 --> DDB

```

## 4. API設計

## 4.1 PSV API

### POST /psv

#### 役割

- CSVアップロード受付
- 非同期処理開始

#### リクエスト

```json
{
  "fileName": "mf.csv"
}
```

#### レスポンス

```json
{
  "psvId": "psv_xxxx",
  "status": "processing"
}
```

---

### GET /psv/{psvId}

```json
{
  "psvId": "psv_xxxx",
  "status": "ready",
  "summary": {}
}
```
