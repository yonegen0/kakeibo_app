# 導入ライブラリ精査一覧

設計思想である「Models・Schemas・Hooks・Componentsの分離」を、MUI（Material UI）の強力なエコシステムに集約して実現するためのライブラリ構成です。

コンポーネントアーキテクチャは **Atom → Molecule → Organism → Template** の階層分離で、各層は独立したドキュメント構造（ファイルコンセプト、Types、Styled Components、ロジック）を持ちます。

## 1. 堅牢なデータ基盤 (Validation & Models)

| ライブラリ名 | カテゴリ | 採用理由 |
| :--- | :--- | :--- |
| **TypeScript** | 言語 | 静的型定義。`Models` 層での Interface 定義に使用し、開発時のバグを激減させます。 |
| **Zod** | バリデーション | スキーマ定義用。APIレスポンスのチェックやCSVバリデーションの要となります。 |

## 2. 非同期通信・状態管理 (Hooks / API)

| ライブラリ名 | カテゴリ | 採用理由 |
| :--- | :--- | :--- |
| **TanStack Query** | Data Fetching | `Hooks` 層の核。データのフェッチ、キャッシュ管理、ロード状態の自動化に必須です。 |
| **Axios** | HTTP Client | API通信用。インターセプターで AWS への認証ヘッダー付与を一元化します。 |
| **AWS SDK (v3)** | Cloud SDK | S3への直接アップロードに必要なモジュール (`@aws-sdk/client-s3`) のみを導入します。 |

## 3. UI・コンポーネント (Components & Design)

| ライブラリ名 | カテゴリ | 採用理由 |
| :--- | :--- | :--- |
| **MUI (Material UI)** | UI Component | `Atoms` や `Molecules` のベース。一貫したデザインシステムを迅速に構築します。 |
| **@mui/icons-material** | Icons | **【活用中】** MUI公式アイコン。Atoms・Organisms 層で使用（`InsightsIcon`・`ErrorOutlineIcon`・`CheckCircleOutlineIcon`・`CloudUploadIcon` など）。複数のコンポーネント層に跨る活用で統一感を実現。 |
| **React Markdown** | Utility | **【活用中】** `MarkdownRenderer`（Atom層）を介して `AIReport`（Organism層）で使用。Bedrock の分析レポートを安全に描画し、テーマに基づいたスタイル適用。 |
| **Recharts** | Chart | `Organisms` で使用。集計データを基に支出推移を可視化します。 |

## 4. 開発・テスト・カタログ化 (DX)

| ライブラリ名 | カテゴリ | 採用理由 |
| :--- | :--- | :--- |
| **Storybook** | UI Catalog | **【本運用】** コンポーネント（Atoms・Organisms・Templates）の独立開発・カタログ化。複数ストーリー定義（AIReport・MarkdownRenderer・TransactionImportTemplate など）で UI パターンの網羅的確認と回帰テストが可能。 |
| **MSW** (Mock Service Worker) | Mocking | API 未完成時でも、`Schemas` に基づくダミーデータを返してフロント開発を止めないために使用します。 |
| **Vitest** | Testing | `Hooks` の計算ロジック（Domain層）が正しく動くかを自動テストします。 |

## 🛠️ インストール実行コマンド

プロジェクト開始時に以下のコマンドを実行して環境を構築します。Tailwind CSS は含まず、MUI の `styled` や `sx` プロップスにスタイリングを統合します。

```bash
# UI関連 (MUI本体, Icon)
npm install @mui/material @emotion/react @emotion/styled @mui/icons-material

# 通信・バリデーション・解析
npm install zod @tanstack/react-query axios react-markdown recharts

# AWS関連 (必要なモジュールのみ)
npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner

# Storybook初期化 (Next.js環境を自動認識)
npx storybook@latest init

# 開発用ツール (Mock, Test)
npm install -D msw vitest @testing-library/react
```

## 💡 コンポーネント設計ベストプラクティス

### 構造化とドキュメント

- **ファイルヘッダー**: `@file` と `@description` で役割を明記
- **セクション区切り**: `/* --- Types --- */` `/* --- Styled Components --- */` などで視認性向上
- **インラインコメント**: JSX各部分の意図を記載（状態ごとの条件分岐など）
- **JSDoc コメント**: 関数・コンポーネントのプロップス、戻り値を明確に記述

### スタイリング手法

- **`styled` 関数**: コンポーネント層で `styled(Paper)`、`styled(Box)` など MUI コンポーネントをカスタマイズ
- **`keyframes` アニメーション**: フェードイン、パルス、スケールなどの動的効果を `styled` に埋め込み
- **グラディエント背景**: `linear-gradient` で奥行き感を演出
- **条件付きスタイル**: `sx` プロップで状態に応じたスタイル切り替え（ホバー、無効化など）
- **トランジション効果**: 状態遷移時の滑らかなアニメーション

### アイコン活用パターン

| 用途 | アイコン | 用途例 |
| :--- | :--- | :--- |
| **情報・洞察** | `InsightsIcon` | AIレポート、分析結果のシンボル |
| **エラー表示** | `ErrorOutlineIcon` | エラーAlert、バリデーション失敗 |
| **成功表示** | `CheckCircleOutlineIcon` | 完了状態、成功フィードバック |
| **ファイル操作** | `CloudUploadIcon` | ファイルアップロード エリア |

### Storybook での充実した検証

各コンポーネント層（Atoms・Organisms など）に対して、複数の状態を表現したストーリーを定義：

- **Loading**: 非同期処理中の状態表示
- **Error**: エラーハンドリングと詳細メッセージ表示
- **Empty**: データなしの空状態ガイダンス
- **Success**: 成功時のコンテンツ表示

これにより、UI パターンの網羅的確認と、修正時の回帰テストが容易になります。
