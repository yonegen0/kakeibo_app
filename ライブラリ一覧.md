# 導入ライブラリ精査一覧 (Next.js / TypeScript)

設計思想である「Models・Schemas・Hooks」の責務分離を支え、サーバーレス環境で効率的に動作するライブラリ群です。

## 1. 堅牢なデータ基盤 (Validation & Models)

| ライブラリ名 | カテゴリ | 軽い説明・採用理由 |
| :--- | :--- | :--- |
| **Zod** | Validation | スキーマ定義用。APIレスポンスのランタイムチェックやCSVバリデーションの要。 |
| **TypeScript** | Language | 静的型定義。`Models` 層での Interface 定義に使用し、開発時のバグを激減させる。 |

## 2. 非同期通信・状態管理 (Hooks / API)

| ライブラリ名 | カテゴリ | 軽い説明・採用理由 |
| :--- | :--- | :--- |
| **TanStack Query** (React Query) | Data Fetching | `Hooks` 層の核。S3/Lambda からのデータフェッチ、キャッシュ管理、ローディング状態を自動化。 |
| **Axios** | HTTP Client | API通信用。インターセプター機能を使って、AWS Gateway への認証ヘッダー付与などを一元化。 |
| **AWS SDK for JS (v3)** | Cloud SDK | クライアントサイドから S3 へ直接アップロード（署名付きURL使用）する際に必要なモジュールのみを導入。 |

## 3. UI・コンポーネント (Components)

| ライブラリ名 | カテゴリ | 軽い説明・役割 |
| :--- | :--- | :--- |
| **MUI (Material UI)** | Component | `Atoms` や `Molecules` のベース。一貫したデザインシステムを迅速に構築。 |
| **Tailwind CSS** | Styling | カスタムコンポーネントの微調整用。Next.js との親和性が高く、軽量なビルドが可能。 |
| **Lucide React** | Icons | 軽量でカスタマイズ性の高いアイコンセット。家計簿のカテゴリ表示などに活用。 |
| **React Markdown** | Utility | `AIReportCard` 用。Bedrock から返却される分析レポート（Markdown）を安全に描画。 |
| **Recharts** | Chart | `Organisms` で使用。DynamoDB から集計したデータを基に、支出推移を可視化。 |

## 4. 開発・テスト支援

| ライブラリ名 | カテゴリ | 軽い説明 |
| :--- | :--- | :--- |
| **MSW** (Mock Service Worker) | Mocking | API が未完成でも、`Schemas` に基づいたダミーデータを返してフロント開発を進めるために使用。 |
| **Jest / Vitest** | Testing | `Hooks` の計算ロジック（Domain層）が正しく動くかを自動テスト。 |

---

## 🛠️ インストール実行コマンド

```bash
# 基本構成
npm install zod @tanstack/react-query axios lucide-react react-markdown recharts

# AWS関連 (必要なモジュールのみ)
npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner

# UI関連
npm install @mui/material @emotion/react @emotion/styled
