# 導入ライブラリ精査一覧

設計思想である「Models・Schemas・Hooks・Componentsの分離」を、MUI（Material UI）の強力なエコシステムに集約して実現するためのライブラリ構成です。

## 1. 堅牢なデータ基盤 (Validation & Models)

| ライブラリ名 | カテゴリ | 採用理由 |
| :--- | :--- | :--- |
| **TypeScript** | 言語 | 静的型定義。`Models` 層での Interface 定義に使用し、開発時のバグを激減させます。 |
| **Zod** | バリデーション | スキーマ定義用。APIレスポンスのチェックやCSVバリデーションの要となります。 |

## 2. 非同期通信・状態管理 (Hooks / API)

| ライブラリ名 | カテゴリ | 採用理由 |
| :--- | :--- | :--- |
| **TanStack Query** | Data Fetching | `Hooks` 層の核。データのフェッチ、キャッシュ管理、ロード状態の自動化に必須です。 |
| **Axios** | HTTP Client | API通信用。インターセプターで AWS への認証ヘッダー付与を一元化します。 |
| **AWS SDK (v3)** | Cloud SDK | S3への直接アップロードに必要なモジュール (`@aws-sdk/client-s3`) のみを導入します。 |

## 3. UI・コンポーネント (Components & Design)

| ライブラリ名 | カテゴリ | 採用理由 |
| :--- | :--- | :--- |
| **MUI (Material UI)** | UI Component | `Atoms` や `Molecules` のベース。一貫したデザインシステムを迅速に構築します。 |
| **@mui/icons-material** | Icons | **【更新】** MUI公式アイコン。カテゴリ表示やボタンに使用し、デザインの統一感を保ちます。 |
| **React Markdown** | Utility | `AIReportCard` 用。Bedrock から返却される分析レポートを安全に描画します。 |
| **Recharts** | Chart | `Organisms` で使用。集計データを基に支出推移を可視化します。 |

## 4. 開発・テスト・カタログ化 (DX)

| ライブラリ名 | カテゴリ | 採用理由 |
| :--- | :--- | :--- |
| **Storybook** | UI Catalog | **【追加】** コンポーネントの独立開発用。MUI のテーマと連携し、UI パターンを網羅的に確認します。 |
| **MSW** (Mock Service Worker) | Mocking | API 未完成時でも、`Schemas` に基づくダミーデータを返してフロント開発を止めないために使用します。 |
| **Vitest** | Testing | `Hooks` の計算ロジック（Domain層）が正しく動くかを自動テストします。 |

---

## 🛠️ インストール実行コマンド

プロジェクト開始時に以下のコマンドを実行して環境を構築します。Tailwind CSS は含まず、MUI の `styled` や `sx` プロップスにスタイリングを統合します。

```bash
# UI関連 (MUI本体, Icon)
npm install @mui/material @emotion/react @emotion/styled @mui/icons-material

# 通信・バリデーション・解析
npm install zod @tanstack/react-query axios react-markdown recharts

# AWS関連 (必要なモジュールのみ)
npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner

# Storybook初期化 (Next.js環境を自動認識)
npx storybook@latest init

# 開発用ツール (Mock, Test)
npm install -D msw vitest @testing-library/react
